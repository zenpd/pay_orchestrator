🔌 API ENDPOINTS

  GET  /regions
       → Returns all 4 available regions

  GET  /regions/{code}/payment-types
       → Returns 5 payment types for region (ZA/US/GB/EU)

  GET  /regions/{code}/rails
       → Returns 8 payment rails for region

  POST /process-payment
       → Process payment with region-specific rail selection
       → Input: region, payment_type, amount, currencies, compliance
       → Output: payment_id, selected_rail, score, cost, processing_time

  GET  /logs/{payment_id}
       → Get all logs for a payment

  DELETE /logs
       → Clear all logs

  GET  /health
       → API health check

💡 EXAMPLE WORKFLOW

  1. User opens Streamlit UI
     └─ UI loads at http://localhost:8501

  2. User selects region: "🇺🇸 United States"
     └─ Frontend dynamically loads US payment types

  3. User selects payment type: "Domestic Wire Transfer"
     └─ UI shows example amount and details

  4. User enters amount: $25,000
     └─ Selects optimization: "Speed-Focused"

  5. User clicks "Process Payment"
     └─ Backend receives request:
        - Region: US
        - Payment Type: Domestic Wire Transfer
        - Amount: 25,000 USD
        - Optimization: Speed-Focused

  6. Backend processes:
     ├─ Validates region (✅ US valid)
     ├─ Loads US configuration
     ├─ Gets US rails (FedWire, RTP, ACH, SWIFT, CHIPS)
     ├─ Calculates score for each rail:
     │  - FedWire: 92.5 (fastest, reliable)
     │  - RTP: 89.3 (instant but less common)
     │  - ACH_Express: 85.2 (slower)
     │  - SWIFT_GPI: 88.0 (international)
     │  - CHIPS: 88.7 (high-value)
     └─ Selects FedWire (highest score)

  7. Response returned to UI:
     ├─ Status: SUCCESS ✅
     ├─ Payment ID: abc123def456
     ├─ Selected Rail: FedWire
     ├─ Route Score: 92.5/100
     ├─ Estimated Cost: $15.00
     ├─ Processing Time: 30 minutes
     └─ Rails Considered: [FedWire, CHIPS, RTP, ...]

  8. UI displays results:
     ├─ Large success banner
     ├─ Metrics cards
     ├─ Rails comparison table
     └─ Execution timeline
